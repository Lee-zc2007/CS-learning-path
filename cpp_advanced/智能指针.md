#智能指针
离开作用域会自动销毁
可以防止内存泄漏，悬空指针，野指针的问题
智能指针本身是一个对象，封装了一个指向另一个对象的指针
标准库中提供了以下智能指针
'''
unique_ptr
shared_ptr
weak_ptr
'''

*unique_ptr*
常用函数
'''
T* get();//获得所管理对象的指针
T* operator->();//调用get也获得了所管理对象的指针，可以间接访问成员
T& operator*();//可以解引用，相当于*get()
T* release();//解除对封装对象的管理，返回封装对象的指针
void reset(T* newObject);//删除原有对象，替换为新的对象
void swap(unique_ptr<T>& other);//与别的unique_ptr交换所管理的对象
'''
与其所管理的对象是一对一的关系（不能有两个unique_ptr对象指向同一个地址）
创建对象
'''
unique_ptr<A> ptr1(new A(参数));
unique_ptr<A> ptr1 = make_unique<A>(类A构造函数的参数);
'''
重要特性：不能被拷贝也不能被赋值（保证封装的指针不和其他对象公用）
但是可以用move对所有权转移(move把左值变成右值 )
'unique_ptr<Rectangle> p2 = move(p1);//将p1中指针的控制权转给p2，p1变成空指针'

*shared_ptr*
多个shared_ptr对象可以管理同一个指针，通过一个共同的引用计数器来管理
计数器为0会将内存释放
常用函数
'''
T* get();
T* operator->();
T& operator*();
void reset(T* newObject);
void swap(unique_ptr<T>& other);
operator bool();
long use_count();//判断有多少shared_ptr共用一个对象
bool unique();//判断use_count是否等于1
static_pointer_cast//对封装指针类型静态转换
dynamic_pointer_cast//动态转换
const_pointer_cast//常量转换
'''
不能循环引用，会导致内存泄漏
shared_ptr内部有两个指针，一个指向动态对象，另一个指向控制块（有use_count和weak_count）

*weak_ptr*
可以解决上述循环引用导致的内存泄漏问题
因为weak_ptr不增加引用计数，指向对象被销毁的时候，自动变成空
必须配合shared_ptr使用
可以将一个shared_ptr作为构造函数的参数初始化或者定义一个空的对象赋值
use_count大于0时所观测对象是可用的
expired为true的时候观测对象是不可用的
weak_ptr同样指向动态对象和控制块
use_count为0的时候会删除动态对象，但是保留控制块对象
