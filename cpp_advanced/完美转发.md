#完美转发
'''
void f(const string& s){
    cout<<"左值"<<s<<endl;
}

void f(string &&s){
    cout<<"右值"<<s<<endl;
}
//分别定义左右值参数的两个重载f

void g(const string& s){
    f(s);
}

void g(string &&s){
    f(s);
}
//定义左右值参数的g，调用f

int main(){
    g(string("Hello1"));//传右值
    string s1("hello2");
    g(s1);//传左值
}
'''
上面的例子中虽然一个传右值，一个传左值，但是在传右值的g中，s是一个具有名称的局部变量，可以获得地址，f(s)还是调用左值的f
为了调用原有的对应函数，可以用static_cast<const string&>和static_cast<string&&>
但是实际操作中，我们一般采用std::forward<>
forward可以保留参数进来的格式，进来是左值就按左值，进来是右值就按右值

*万能引用*
当参数数量增加的时候，重载函数的数量会几何级增长，因此我们需要用函数模板来替代
'''
template<class T>
void g(T&& v){
    f(std::forward<T>(v));
}
'''
这里的T&&是万能引用（只有模板参数的右值引用形式才是万能引用，不是所有的右值引用都是万能引用）
万能引用既可以绑定左值也可以绑定右值
在函数模板中，只有T&&这种形式作为形参才能叫万能引用

*引用折叠*
左值引用优先级更高，除了T&& &&折叠为&&，其他三种都折叠为T&
