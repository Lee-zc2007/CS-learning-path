#异常
用于处理程序运行中的特殊情况，如文件无法打开等
包含try catch throw三个部分

'''
float divide(float a, float b){
    if (b==0)
        throw runtime_error("Divided by zero!")
    return a/b;
}

int main(){
    try{
        int c = divide(3.1,0);
        cout<<c<<endl;
    }
    catch(runtime_error& e){
        cout<<"exception:"<<e.what()<<endl;
    }
    return 0; 
}
'''
runtime_error是一个标准库里定义的异常类

cpp中可以将各种类型的变量 对象 指针作为异常抛出
推荐自定义或继承标准库中的异常类std::exception

一个try代码块可以跟多个catch代码块
越通用的异常越放在后面，例如继承类放在基类前面

对于未知异常可以用catch(...)来捕获，通常不建议直接保留，应该再抛出，在这部分代码块加上throw;即可

*Resource Acquisition Is Initialization(RAII)资源获取即初始化*
对于动态分配的内存或文件句柄等，通常使用类对这些资源封装，在自动创建的类对象自动释放的时候会调用类的析构函数释放动态内存

对于动态内存，通常使用shared_ptr进行封装
如'shared_ptr<MyResource> pRes(new MyResource("[doSomething]))'
当pRes被销毁时，指针申请的内存会自动释放
