#右值引用和移动语义
左值也被叫做locator value
能够被引用或者取地址的表达式是左值
有些左值是不可修改的，比如const
‘int *p = &c;//这里的c也是左值’
c++11后，表达式分为glvalue和rvalue，前者包括lvalue和xvalue，后者包括xvalue和prvalue

*lvalue*
'''
int a=8;
double m = 1.5;

int *p = new int[5];
*p = 4;

int b[5];
b[1] = 10;

int& r = a;//左值引用的赋值运算符右侧必须是个左值
'''
此外，类对象的成员变量是个左值
返回引用的函数的调用表达式的调用是左值
'''
int &refNumeber()
{
    static int n = 1;
    return n;
}

int main()
{
    refNumber() = 5;//相当于给n赋值5
}
'''

*rvalue*
非静态类成员函数
枚举
数组
this指针
lambda表达式
一些内置运算符的表达式 a++, a && b, a+b等等

*右值引用*
'''
int &&c = 5;//右值引用只能绑定右值
'''
在定义函数的时候可以将左值参数和右值参数分开，分别重载

*移动语义*
定义一个类没有定义拷贝构造函数可能会导致浅拷贝
需要拷贝复制运算符重载函数（传递左值引用）
还要实现移动构造函数和移动赋值运算符重载函数（传右值引用且不需要const限制）
拷贝赋值：对象已经存在，用另一个对象的内容覆盖它
移动构造：创建新对象，接管临时对象的资源
移动赋值：对象已经存在，接管临时对象的资源

*copy elision 拷贝省略*
'CharBuffer buffer1(CharBuffer(100))'
编译器没有先创建临时对象，然后调用移动构造函数，直接调用了构造函数
编译器再一些条件下会省略临时对象创建这一步，直接在内存位置创建对象
cpp11是可以通过调整编译器参数决定是否优化，17以后则是默认优化了，可以手动设置为11

'return value optimization(RVO)返回值优化'
'''
CharBuffer generate(int n){
    return CharBuffer(n);
}

int main(){
    CharBuffer buffer2 = generate(100);
}
'''
类似copy elision，同样省略了中间步骤，直接构造buffer2

'named return value optimization(NRVO)具名的返回值优化'
'''
CharBuffer generate_nv(int n){
    CharBuffer buf(n);
    return buf;
}

int main(){
    CharBuffer buff2 = generate_nv(100);
}
'''
这里buf和buff2的地址相同，编译器省略步骤，直接在buff2的位置构造对象，所以两者其实是同一个对象，离开函数作用域的时候只有buf的名字销毁了，这个对象的生命周期只进行了一次析构

*移动构造和移动赋值的使用*
std::move(buff1)//相当于static_cast<CharBuffer&&>(buff1);
将左值转为右值引用
'''
int main(){
    CharBuffer buff2(10);
    {
        CharBuffer buff1(100);
        buff2 = std::move(buff1);
    }
}
'''
通过将即将销毁的buff1转为右值引用，从而将其资源回收再利用
遇到拷贝构造涉及到深拷贝，需要复制较多数据可以采用移动语义
智能指针unique_ptr对所管理对象有唯一的控制权，转移控制权的时候也需要移动语义

*xvalue将亡值*
函数返回的右值引用 std::move()
左值到右值的静态转换 static_cast<T&&>(v)
一个未命名的右值对象获得它的非静态成员变量的表达式
